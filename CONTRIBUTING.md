# Contributing to scope-lock

Thanks for your interest in contributing! Scope Lock is a small, focused project and contributions are welcome.

## Ways to contribute

- **Bug reports** — if the skill misbehaves during a session, open an issue with what you expected vs. what happened
- **Feature requests** — open an issue describing the use case, not just the feature
- **Documentation** — README improvements, typo fixes, usage examples
- **Code** — fork the repo, make your changes on a branch, and open a PR

## Before opening a PR

1. Keep changes focused — one PR, one concern
2. If it's a significant change, open an issue first so we can discuss the approach
3. Test the skill in a real Claude Code session before submitting

## Style

- Commit messages: `type: description` (e.g., `fix: correct scope check logic`, `docs: add usage example`)
- Keep SKILL.md readable — it's consumed by an LLM, so clarity matters more than brevity

## Questions?

Open a [Discussion](https://github.com/Ktulue/scope-lock/discussions) or reach out on [Ko-fi](http://ko-fi.com/ktulue).
